# Repository Strategy

RidgeLink Technologies repositories should be organized like a real engineering organization.

## Public repositories

- `.github`
- `website`
- `docs`
- `homelab`
- `powershell`
- `automation`
- `terraform`
- `azure`
- `architecture`
- `templates`
- `blog`

## Private repositories

- `branding`
- `personas`
- `design-system`
- `marketing-assets`
- `internal-notes`

## Naming standards

Use lowercase repository names and avoid unnecessary prefixes inside the organization.
